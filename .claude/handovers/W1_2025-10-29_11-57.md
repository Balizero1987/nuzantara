### Problem Identified

**Symptoms:**
- FLAN Router auto-stopping after 1 minute idle
- Health check: `"flanRouter": "unhealthy"`

- Error: `"instance refused connection"`

**Root Causes:**
1. `auto_stop_machines = 'stop'` â†’ Machine stops after 1 min idle
2. `min_machines_running = 0` â†’ Every request triggers cold start
3. No health check grace period â†’ Default timeout too short for model loading
4. Health endpoint too strict â†’ Returns error during initialization

### Solutions Applied

#### 1. Fly.io Configuration (fly.toml)
```toml
[http_service]
  internal_port = 8000
  force_https = true
  auto_stop_machines = 'off'        # â† CHANGED: Disabled auto-stop
  auto_start_machines = true
  min_machines_running = 1          # â† CHANGED: Keep 1 machine running
  processes = ['app']

  # â† ADDED: Health check configuration
  [[http_service.checks]]
    grace_period = "90s"            # Give 90s for model to load
    interval = "30s"
    method = "GET"
    timeout = "10s"
    path = "/health"
```

#### 2. Health Endpoint Improvement (router_only.py)
```python
@app.get("/health")
async def health_check():
    """
    Liveness check - always responds 200 OK
    Even during initialization to prevent Fly.io from killing machine
    """
    if not router:
        return {
            "status": "initializing",
            "model": "loading...",
            "ready": False  # â† New flag
        }

    return {
        "status": "healthy",
        "model": router.model_name,
        "ready": True
    }

@app.get("/ready")
async def readiness_check():
    """Readiness check - returns 503 if not ready"""
    if not router:
        raise HTTPException(status_code=503)
    return {"status": "ready"}
```
